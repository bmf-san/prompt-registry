# Review: ドキュメントレビュー

文章・ドキュメントをレビューする際のチェックリスト。文章の種類や目的に応じて項目を選択して使用する。

## チェックリスト

### 全体構成レベル

#### 構造
- [ ] 見出し構成は論理的か（階層構造が適切か）
- [ ] 見出しだけで要旨が掴めるか
- [ ] 目次があれば、それを見ただけで内容が推測できるか
- [ ] 情報の順序は適切か（時系列、重要度順、問題→解決など）
- [ ] 導入と結論は明確か
- [ ] 全体の分量は適切か（長すぎ/短すぎないか）

#### 論理性
- [ ] 主張と根拠の関係は明確か
- [ ] 論理の飛躍はないか
- [ ] 因果関係は正確に説明されているか
- [ ] 反論への配慮はあるか（必要な場合）
- [ ] 結論は本文の内容を適切に要約しているか

#### 読者への配慮
- [ ] 読者が最初に知りたいことが冒頭で示されているか
- [ ] 読者の前提知識を適切に想定しているか
- [ ] 読者の疑問が生じるタイミングで説明があるか
- [ ] 読者のモチベーション（なぜ読むべきか）が示されているか

### 内容レベル

#### 正確性・信頼性
- [ ] 事実関係は正確か
- [ ] データや統計は信頼できるソースから引用されているか
- [ ] 引用元は明示されているか
- [ ] 日付や数値は最新か
- [ ] 主観と客観が混同されていないか
- [ ] 断定すべきでないことを断定していないか

#### 十分性
- [ ] 判断理由や背景が説明されているか
- [ ] 重要な用語は定義されているか
- [ ] 抽象的な説明に具体例が伴っているか
- [ ] 図表は必要に応じて使われているか
- [ ] 代替案や比較があれば、それらが説明されているか
- [ ] 読者が次のアクションを取れる情報があるか（必要な場合）

#### 簡潔性
- [ ] 重要なポイントに焦点が絞られているか
- [ ] 本題から逸れた話はないか
- [ ] 同じ内容が繰り返されていないか
- [ ] 一つの情報を複数の方法で説明しすぎていないか

### 段落レベル

#### 構造
- [ ] 各段落に一つの主題があるか
- [ ] 段落の冒頭でトピックが明確か
- [ ] 段落間の繋がりは明確か（接続詞、指示語の使用）
- [ ] 段落の長さは適切か（目安: 3〜7文程度）

#### 内容
- [ ] 段落内の文の順序は論理的か
- [ ] 具体例は適切な位置に配置されているか
- [ ] 一般論と具体例のバランスは取れているか

### 文レベル

#### 構造
- [ ] 一文が長すぎないか（目安: 60〜80字以内）
- [ ] 主語と述語の関係は明確か
- [ ] 修飾語と被修飾語の関係は明確か
- [ ] 修飾語の位置は適切か（係り受けが明確か）
- [ ] 受動態の過度な使用はないか

#### 明瞭性
- [ ] あいまいな表現はないか（「〜など」の多用）
- [ ] 二重否定で読みにくくなっていないか
- [ ] 指示語（これ、それ、あれ）の指す対象は明確か
- [ ] 文末表現は統一されているか（である調/ですます調）
- [ ] 一文に複数の主張が詰め込まれていないか

#### 冗長性
- [ ] 冗長な表現はないか（「〜することができる」→「〜できる」）
- [ ] 不要な形容詞・副詞はないか
- [ ] 同じ意味の言葉が重複していないか

### 用語・表現レベル

#### 用語
- [ ] 専門用語は読者レベルに適切か
- [ ] 初出の専門用語は説明されているか
- [ ] 用語の使い方は統一されているか
- [ ] 業界用語や略語は適切に使われているか
- [ ] カタカナ語は必要な場合のみ使われているか

#### 表記
- [ ] 誤字脱字はないか
- [ ] 表記ゆれはないか（漢字/ひらがな、送り仮名）
- [ ] 数字の表記は統一されているか（全角/半角、算用数字/漢数字）
- [ ] 英数字・記号の前後のスペースは統一されているか
- [ ] 句読点の使い方は適切か

#### トーン・スタイル
- [ ] 文体は読者と目的に適切か（フォーマル/カジュアル）
- [ ] トーンは全体で一貫しているか
- [ ] 敬語の使い方は適切か（必要な場合）
- [ ] 視点（一人称/三人称）は統一されているか

### 技術文書特有のチェック（該当する場合）

#### コード・技術要素
- [ ] コード例は正確で動作するか
- [ ] コードには適切なコメントがあるか
- [ ] 技術仕様は最新版に基づいているか
- [ ] 環境依存の情報は明記されているか
- [ ] エラーハンドリングやエッジケースへの言及があるか

#### 図表
- [ ] 図表にはキャプションがあるか
- [ ] 図表は本文で参照されているか
- [ ] 図表の品質は十分か（解像度、見やすさ）
- [ ] アーキテクチャ図は最新の状態を反映しているか

### 最終チェック

#### 全体の印象
- [ ] タイトルは内容を適切に表しているか
- [ ] 概要・要約は内容を適切に反映しているか
- [ ] 全体を通して読みやすいか
- [ ] 読後に読者が得られる価値は明確か
- [ ] 次のアクションや参考情報が示されているか（必要な場合）

#### メタ情報
- [ ] 著者名は記載されているか（必要な場合）
- [ ] 作成日・更新日は記載されているか
- [ ] 対象バージョンや前提条件は明記されているか（技術文書の場合）
- [ ] 参考文献・関連リンクは適切か
