# Architecture Strategy Review Checklist

アーキテクチャ戦略を策定・レビューする際に確認すべき項目。

## ビジネスとの接続

- [ ] ビジネス目標が明確に理解されているか
- [ ] 戦略がビジネス目標に貢献することが説明できるか
- [ ] ステークホルダーの期待値が把握されているか

## Why（なぜ）の明確化

- [ ] 各原則に「なぜ」が明記されているか
- [ ] To-Beに「なぜその姿を目指すのか」が明記されているか
- [ ] 手段が目的化していないか

## What（何を）の明確化

- [ ] 達成すべきゴールが具体的か
- [ ] 成功指標（KPI）が設定されているか
- [ ] 測定可能な目標になっているか

## 賞味期限の設定

- [ ] 設計の賞味期限が明示されているか
- [ ] 事業・組織・プロダクト・技術の4視点で検討されているか
- [ ] 期間設定の根拠が説明できるか

## 原則（基本方針）

- [ ] 原則は3〜5個に絞られているか
- [ ] 各原則にWhy & Whatが含まれているか
- [ ] トレードオフが明示されているか
- [ ] 誰もが理解できる明快さがあるか

## To-Be（目指す姿）

- [ ] Why & Whatの形式で記述されているか
- [ ] 手段（How）が混入していないか
- [ ] 現実的で実現可能な姿か
- [ ] 賞味期限内に到達可能か

## ギャップ分析

- [ ] 現状の課題が明確に特定されているか
- [ ] 優先順位付けの基準が明確か
- [ ] ビジネスインパクトが評価されているか
- [ ] 実現難易度が評価されているか

## ガバナンス

- [ ] 意思決定プロセスが定義されているか
- [ ] 定期的な見直しサイクルが設定されているか
- [ ] ステークホルダーとの共有方法が明確か

## 次のステップ

- [ ] System-Architectへの引き継ぎ事項が明確か
- [ ] App-Architectへの引き継ぎ事項が明確か
- [ ] 実行計画の方向性が示されているか
