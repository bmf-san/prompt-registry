# Review: コードレビュー

実装・コードレビュー時に確認すべきチェックリスト。

## チェックリスト

### 基本
- [ ] コードは正しく動作するか
- [ ] エッジケースに対応しているか
- [ ] エラーハンドリングは実装されているか
- [ ] 既存のコーディング規約に従っているか

### 命名
- [ ] 変数名・関数名は意図を明確に表しているか
- [ ] マジックナンバーは定数化されているか
- [ ] 略語は適切に使われているか

### 構造
- [ ] 関数は適切な長さか（目安: 50行以内）
- [ ] 1関数1責務の原則を守っているか
- [ ] 重複コードはないか
- [ ] ネストは深すぎないか（目安: 3段階以内）

### パフォーマンス
- [ ] 計算量は許容範囲か
- [ ] 不要なループや処理はないか
- [ ] データベースのN+1問題はないか
- [ ] メモリリークの可能性はないか

### セキュリティ
- [ ] ユーザー入力は検証されているか
- [ ] SQLインジェクションの可能性はないか
- [ ] XSSの可能性はないか
- [ ] 認証・認可は最小権限の原則に従って実装されているか

### テスタビリティ
- [ ] 依存関係は注入可能か
- [ ] 副作用は最小化されているか
- [ ] テストしやすい構造か

### 可読性
- [ ] コメントにはなぜそうしたかを説明しているか
- [ ] コードの意図は明確か
- [ ] 複雑なロジックには説明があるか
