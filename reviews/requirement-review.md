# Requirement Review Checklist

要件を定義・レビューする際に確認すべき項目。

## 課題の明確性

- [ ] 解決すべき課題が明確に定義されているか
- [ ] なぜその課題を解決する必要があるか説明できるか
- [ ] 課題の影響範囲（誰が困っているか）が特定されているか
- [ ] 課題の発生頻度・深刻度が把握されているか

## ゴールの具体性

- [ ] 成功の定義が明確か
- [ ] 測定可能なKPIが設定されているか
- [ ] ゴール達成の期限が設定されているか
- [ ] ゴールは現実的か（達成可能か）

## ユーザーストーリー

- [ ] すべてのストーリーに「As a / I want to / So that」が含まれているか
- [ ] 受け入れ基準が具体的か
- [ ] ストーリーのサイズは適切か（大きすぎないか）
- [ ] ストーリー間の依存関係は明確か

## 優先順位

- [ ] すべての要件に優先順位が付けられているか
- [ ] 優先順位の判断基準は明確か
- [ ] Must Have の要件だけでMVPとして成立するか

## スコープ

- [ ] スコープ外の項目が明示されているか
- [ ] スコープ変更の判断基準が定義されているか

## リスク

- [ ] 主要なリスクが特定されているか
- [ ] 各リスクへの対応策が定義されているか

## ステークホルダー合意

- [ ] ユーザー・顧客の要件との整合性が確認されているか
- [ ] ビジネス側の期待値と整合しているか
- [ ] 開発チームの実現可能性が確認されているか
