# System Design Review Checklist

システム設計をレビューする際に確認すべき項目。

## 要件との適合性

- [ ] すべての機能要件を満たしているか
- [ ] 非機能要件（パフォーマンス、可用性など）を満たしているか
- [ ] 制約条件（予算、時間、技術）の中で実現可能か

## アーキテクチャの品質

- [ ] 責務が明確に分離されているか
- [ ] コンポーネント間の結合度は低いか
- [ ] コンポーネント内の凝集度は高いか
- [ ] 単一障害点はないか
- [ ] スケーラビリティは確保されているか

## データ設計

- [ ] データモデルは正規化されているか（必要に応じて）
- [ ] インデックス設計は適切か
- [ ] データの整合性は保証されているか
- [ ] バックアップ・リカバリ戦略はあるか

## セキュリティ

- [ ] 認証・認可の仕組みは適切か
- [ ] データの暗号化は適切か
- [ ] OWASP Top 10への対策はあるか
- [ ] 監査ログは設計されているか

## 運用・保守

- [ ] ログ設計は適切か
- [ ] 監視・アラートの仕組みはあるか
- [ ] デプロイ戦略は明確か
- [ ] ロールバック手順は定義されているか
- [ ] ドキュメントは十分か

## 将来性

- [ ] 新機能の追加は容易か
- [ ] 技術的負債の蓄積を防げるか
- [ ] 段階的な移行は可能か（既存システムがある場合）
