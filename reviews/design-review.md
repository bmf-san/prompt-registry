# Design Review Checklist

デザインを作成・レビューする際に確認すべき項目。

## ユーザビリティ

- [ ] ユーザーのゴールが明確か
- [ ] タスクを完了するまでのステップ数は最小か
- [ ] 各ステップで次にすべきことが明確か
- [ ] 誤操作を防ぐ仕組みがあるか
- [ ] エラーからの回復方法が明確か

## 視認性・可読性

- [ ] 重要な情報が目立っているか
- [ ] テキストは読みやすいサイズか（最小14px推奨）
- [ ] コントラスト比は十分か（WCAG AA基準: 4.5:1以上）
- [ ] 色だけに依存した情報伝達をしていないか

## インタラクション

- [ ] タップ/クリック可能な要素がわかりやすいか
- [ ] ボタンサイズは十分か（モバイル: 最小44x44px）
- [ ] ホバー、フォーカス、アクティブ状態が設計されているか
- [ ] ローディング状態が示されているか
- [ ] フィードバックが適切に提供されているか

## 一貫性

- [ ] デザインパターンが統一されているか
- [ ] 用語が統一されているか
- [ ] アイコンの使い方が一貫しているか
- [ ] スペーシングが一貫しているか

## アクセシビリティ

- [ ] キーボード操作に対応しているか
- [ ] スクリーンリーダーに対応しているか
- [ ] 色覚異常への配慮があるか
- [ ] アニメーションの停止オプションがあるか（必要な場合）

## レスポンシブ

- [ ] 主要なデバイスサイズで動作するか
- [ ] タッチとマウス両方に対応しているか
- [ ] 横持ち・縦持ち両対応か（必要な場合）

## エッジケース

- [ ] データがない場合の表示が設計されているか
- [ ] エラー時の表示が設計されているか
- [ ] ローディング時の表示が設計されているか
- [ ] 権限不足時の表示が設計されているか
