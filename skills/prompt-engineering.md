# プロンプトエンジニアリングスキル

プロンプト設計のフレームワーク・パターンカタログ。タスクの性質に応じて適切な構造とパターンを選択するための参照資料。

## プロンプト設計フレームワーク

プロンプト全体を構造化するためのテンプレート。基本パターンを組み合わせる際の骨格として使う。

| フレームワーク | 構成要素 | 向いているケース |
|---|---|---|
| **COSTAR** | Context / Objective / Style / Tone / Audience / Response | 汎用。出力の制御性を高めたいとき |
| **CREATE** | Character / Request / Examples / Adjustment / Type / Extras | 専門家ペルソナを使った分析・創造的タスク |
| **KERNEL** | Keep it simple / Easy to verify / Reproducible / Narrow scope / Explicit constraints / Logical structure | トークン削減と精度を両立したいとき |

## プロンプティング・パターンカタログ

### 基本パターン

| パターン | 概要 | 向いているケース |
|---|---|---|
| **ゼロショット** | 例を一切与えず、そのまま質問・指示する | 汎用的なタスク、シンプルな質問 |
| **ワンショット** | 例を1つだけ示してから本題を聞く | 出力の形式を軽くそろえたいとき |
| **Few-shot** | 例を複数（2〜10件程度）示してから本題を聞く | 出力フォーマットや判断基準を固定したいとき |
| **役割プロンプティング** | 「あなたは〇〇の専門家です」と役割を与える | 特定ドメインの深い知識・語彙を引き出したいとき |
| **コンテキストプロンプティング** | 背景・制約・状況を詳しく設定してから指示する | モデルの回答範囲を絞り、的外れな回答を防ぎたいとき |
| **メタプロンプティング** | モデル自身に「良いプロンプトを設計させる」 | プロンプトを改善・自動生成させたいとき |

### 推論・思考パターン

| パターン | 概要 | 向いているケース |
|---|---|---|
| **Zero-shot CoT** | 「ステップバイステップで考えて」と一言追加する | 手軽に推論精度を上げたいとき |
| **Few-shot CoT** | 推論過程ごと例示を与える | モデルに倣わせたい論理パターンがある場合。⚠️ GPT-4o・Claude 3.5 などの最新推論モデルでは、例示がかえってモデルの内部推論を妨げる場合があるため注意 |
| **Self-Consistency** | 同じ質問を複数回投げて、多数決で最終回答を決める | 算術・論理問題で精度を高めたいとき |
| **Least-to-Most（LtM）** | 問題をサブ問題に分解し、順番に解かせる | 複雑な多段階問題、長文推論 |
| **Tree of Thoughts（ToT）** | 複数の思考の「枝」を生成し、最良の経路を探索させる | 戦略立案・不確実性の高い意思決定 |
| **ReAct（Reason + Act）** | 推論（思考）と行動（ツール呼び出し・検索）を交互に繰り返す | 外部ツールやAPIを使うエージェント型タスク |

### 検証・品質担保パターン

| パターン | 概要 | 向いているケース |
|---|---|---|
| **Generated Knowledge（GKP）** | 回答する前に関連知識をモデル自身に書き出させ、それを根拠として使う | 常識推論・科学的質問への回答精度を上げたいとき |
| **Self-Refine** | 初期回答 → 自己フィードバック → 修正、を繰り返す | コード品質向上・文章のブラッシュアップ |
| **Reflexion** | 自己批判のループで失敗を検出・修正し続ける | コード生成・推論エラーの自己修正 |
| **Chain of Verification（CoVe）** | 回答内の事実を検証する質問を別途生成・回答させる | 事実の正確性が最優先のタスク |

## タスク別の選び方

| タスクの性質 | 推奨パターン |
|---|---|
| シンプルな質問・汎用タスク | ゼロショット |
| 出力フォーマットをそろえたい | Few-shot |
| 専門知識を引き出したい | 役割プロンプティング + COSTAR |
| 多段階の算術・論理問題 | Chain-of-Thought（CoT） |
| 精度をとことん高めたい算術問題 | CoT + Self-Consistency |
| 複雑な構造を持つ難問 | Least-to-Most（LtM） |
| 外部ツール・検索が必要なタスク | ReAct |
| 戦略策定・オープンエンドな問題 | Tree of Thoughts（ToT） |
| 事実の正確性が最優先 | Chain of Verification（CoVe） |
| コード・文章の品質改善 | Self-Refine |
| 推論エラーの自己修正 | Reflexion |
