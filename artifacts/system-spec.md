# Artifact: System Spec

システムの仕様を記録・共有するためのドキュメント構成テンプレート。
新規プロジェクトの設計フェーズだけでなく、既存システムの仕様理解・メンタルモデルの形成にも使える。

**使い分けの目安**:
- **Design Doc**: 「どう作るか」の意思決定フェーズに使う（新規設計・変更の合意）
- **System Spec**: 「どうなっているか」の記録フェーズに使う（現状の仕様・構造の文書化）

## ディレクトリ構成

```
docs/spec/
├── 01_overview.md           # 全体概要
├── 02_system_architecture.md # システム構成
├── 03_data_model.md         # データモデル
```

## 書き方のガイドライン

### どのセクションから始めるか

全ファイルを一度に書く必要はない。
**推奨する優先順位**: `01_overview.md` → `02_system_architecture.md` → `03_data_model.md` → それ以外

### メンテナンス可能な形にする

- アーキテクチャ図は Mermaid などコードで表現する（画像は更新されなくなりやすい）
- 更新方針を `01_overview.md` に明記し、鮮度が落ちた状態を放置しない

### ドキュメントの複雑さはシステムの複雑さを映す

メンテナンスが困難になる場合、ドキュメントの書き方の問題ではなくシステムの複雑性の問題である可能性がある。その気づきの場としても活用できる。
